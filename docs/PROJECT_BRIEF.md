# AntColonyAI — Project Brief

> Онлайн-игра-наблюдение про муравьёв. Дух: SimAnt + Agar.io + тамагочи + idle-симулятор + песочница.
> Исторический бриф раннего MVP. Он полезен для понимания исходной идеи, но уже не является
> единственным источником правды по текущей архитектуре.
>
> Актуальное состояние проекта см. в `README.md`, `docs/Документация.md`,
> `docs/Планы на будущее.md` и `docs/Оптимизация.md`.

---

## 1. Суть идеи

Игрок почти не управляет муравьями напрямую. Он получает/покупает королеву или стаю, запускает её на общую карту и наблюдает.

Муравьи живут сами:
- ищут еду, строят муравейник, размножаются, воюют, умирают, захватывают территорию.

Игрок только наблюдает и иногда вмешивается:
- подкидывает еду, покупает угрозы/события, улучшения среды, запускает новую колонию.

Ключевая фишка: **симуляция идёт на сервере даже офлайн**. Зашёл на следующий день — колония выросла, вымерла или захватила карту.

База для переиспользования: у брата есть браузерный прототип в стиле Agar.io (боты-кружки, классы, лидерборд, наблюдение за AI) — потенциально подходит как основа рендера/онлайна.

---

## 2. Главный вывод

Проект реалистичен для MVP, **если не считать честно тысячи муравьёв по всей карте**.
Вся сложность — в производительности офлайн-мира. Архитектуру строим вокруг этого.

---

## 3. Решение по ИИ — гибрид

- **Муравей = простой агент.** State-machine (`search → carry → return → fight → idle/death`) + реакция на феромоны. Без «сознания», без A*-пути на каждого.
- **Колония = «умный» слой.** Агрегаты (еда, население, нехватка ролей), регулирует пропорции рождения и приоритеты развития. Это коллективный разум.
- **LLM — только асинхронно/батчем** для текста событий и «дневника колонии». **Никогда** как мозг муравья.
- **Феромоны = сетка (grid)** со значениями, испарением и диффузией. **Не** объекты у каждого муравья.

---

## 4. Ключевой приём — LOD симуляции

- **Зона с наблюдателем** → детально считаем каждого муравья.
- **Зона без наблюдателей** → агрегатная симуляция: колония = набор чисел (еда, население, прирост) + редкие случайные события.

Так мир живёт офлайн без адских нагрузок.

---

## 5. Стек MVP

| Слой | Решение | Примечание |
|------|---------|------------|
| Клиент | Браузер, PixiJS (Canvas/WebGL) + TypeScript | Только рендер |
| Сервер | Node.js + TypeScript, авторитетная симуляция | Горячий цикл позже при нужде → Go/Rust |
| Транспорт | WebSocket | Клиенту шлём только видимую область |
| Состояние | Redis (живое состояние + снапшоты) + PostgreSQL (игроки, валюта, метаданные) | На самом первом шаге допустим JSON-файл |
| Тик | Фиксированный серверный тик 5–10/сек | Отдельно от частоты кадров клиента |

> Перед фиксацией стека уточнить, на чём прототип брата — это влияет на выбор клиента.

---

## 6. Карта — две слоя (surface + underground)

Минимальная двухслойная архитектура. Слои асимметричны:

- **Surface (поверхность):** 2D top-down, грид 100×100, тайлы/чанки. Здесь живут феромоны и весь поиск еды.
- **Underground (муравейник):** НЕ грид и НЕ тоннели. Простая сцена с фиксированными слотами:
  матка, кладка яиц, склад еды, вход/выход. Феромонов под землёй нет, пути не считаем.

**Связь слоёв — одна точка:** `entrance` (вход в муравейник). Муравей доходит до входа на одном
слое → переносится на вход другого слоя. Геометрию перехода не рисуем.

> Полноценное подземелье (тоннели, строительство, side-view 2D-срез) — отложено. В MVP underground
> — это минимальная комната-сцена.

## 7. Структура данных (черновик)

```ts
type Colony = {
  id;
  ownerId;
  food;
  population: { workers; soldiers; scouts; eggs };
  queenAlive;
  nestCapacity;
  territory: Set<TileId>;
  detailLevel: "full" | "aggregate";
};

type Ant = {  // только в full-зонах
  id;
  colonyId;
  role: worker | soldier | scout;
  layer: "surface" | "underground";   // на каком слое находится муравей
  state: search | carry | return | fight | idle;
  pos;        // координата в пространстве текущего слоя
  energy;
  target?;
};

type PheromoneGrid = Float32Array;   // только на surface, значение на тайл

// Underground = сцена с фиксированными слотами, НЕ грид
type Underground = {
  queen: { pos; alive };
  eggs: { pos; maturity }[];
  foodStorage: number;
  entrance: pos;                       // общая точка перехода surface↔underground
  ants: AntId[];                       // кто сейчас под землёй
};

type FoodSource = { pos; amount };
type Enemy = { id; type: beetle | spider; pos; hp };
```

## 8. Серверный цикл (псевдокод)

```
каждый тик:
  для колонии:
    если full:
      для муравья → state-machine, движение, energy, еда, бой, смерть
      обновить феромонную сетку (испарение + диффузия)
    если aggregate:
      food += приток
      population += прирост - смертность
      изредка событие
    смерть королевы / голод
    рождение по пропорциям
  раз в N тиков → снапшот в Redis
  наблюдателям → только их видимая область
```

## 9. Механики в первую версию

- Старт королевы → рождение рабочих.
- Феромонный поиск еды и перенос в гнездо.
- Голод и смерть.
- Наблюдение за колонией.
- Лог событий.

## 10. Отложить

- Несколько колоний и войны.
- Солдаты / разведчики как роли.
- Подземный срез.
- Враги (жуки / пауки).
- Онлайн на несколько игроков.
- Донат.

---

## 11. Донат (когда дойдём, без «казино»)

- Никакого рандома за деньги, никакого pay-to-win.
- Звёзды по фикс-цене: косметика, новый слот колонии, разовый корм, события-угрозы как контент, мягкие ускорители с кулдауном.
- Потолок роста задаёт механика, не кошелёк.

---

## 12. Онлайн (позже)

- Один серверный мир = одна авторитетная симуляция.
- Игроки получают слот, чужие колонии read-only, общий лог событий.
- Добавляем после стабильного одиночного ядра.

---

## 13. Риски

1. **Производительность офлайн-мира (главный)** → LOD + агрегаты.
2. **Скука от пассивности** → заметные события, фидбек «пока тебя не было…».
3. **Баланс эмерджентного поведения** → долгий тюнинг параметров.
4. **Баланс доната** → не скатиться в pay-to-win.
5. **Соблазн сразу MMO** → сначала один игрок, одна колония.

---

## 14. Первый шаг

Один сервер, одна колония, два слоя (surface + минимальный underground), феромонный поиск еды.
Без онлайна, врагов, тоннелей, доната.

**Старт игры — под землёй:** матка, рядом несколько рабочих и несколько яиц. Матка — центр колонии:
пока жива → копит ресурс развития со склада еды, кладёт яйца, яйца вызревают в рабочих.

**Цикл рабочего (по слоям):**
- underground: от матки/склада → к входу (`entrance`) → переход на surface;
- surface: `search` (по градиенту феромона еды) → нашёл еду → `carry` (оставляет феромон) → к входу → переход в underground;
- underground: `deposit` на склад → `idle` → снова к выходу;
- любой слой: `death` от голода/энергии.

Конкретика MVP (по проектным указаниям):
- surface: грид 100×100, несколько источников еды, феромоны (испарение + диффузия);
- underground: сцена с фиксированными слотами (матка, яйца, склад, вход) — без грида и тоннелей;
- одна точка перехода `entrance` связывает слои;
- сервер считает симуляцию, клиент рисует и переключает вид Surface / Underground;
- состояние мира — JSON-файл (Redis/Postgres позже).
