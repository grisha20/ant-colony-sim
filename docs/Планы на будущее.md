# Планы на будущее — AntColonyAI

> Дорожная карта со сквозной нумерацией. У каждого пункта статус:
> **[Выполнено]** — сделано и работает; **[Частично]** — есть основа, нужна доработка;
> **[Не начато]** — ещё не делали.
>
> Принципы: маленькие шаги; муравей простой, «ум» — на уровне колонии; перед большими
> изменениями — оценка рисков и план по файлам; логику и рендер держать раздельно.

---

## Ядро симуляции

1. **[Выполнено]** Сервер-авторитет + PixiJS-клиент, два слоя (surface/underground), WebSocket, тик 10/сек.
2. **[Выполнено]** Муравей = конечный автомат (FSM) + реакция на феромоны.
3. **[Выполнено]** Феромоны = две сетки `Float32Array` на surface (испарение + диффузия).
4. **[Выполнено]** Подземелье: 3 камеры + граф путей (`nav.ts`), движение по тоннелям.
5. **[Выполнено]** Полная цепочка расплода: матка кладёт → няньки несут яйцо в ясли → личинка → кормление → рабочий.
6. **[Выполнено]** Матка: кладка, питание со склада, смерть от голода.
7. **[Выполнено]** Модель питания «жизнь от склада»: рабочие едят из резерва, пока он не пуст.
8. **[Выполнено]** Сохранение мира в `snapshot.json`; bat-файлы (Запуск/Стоп/Рестарт/Сброс).
9. **[Выполнено]** Пиксельный рендер + оптимизация (кэш текстур, пулы спрайтов, статичные слои).
10. **[Выполнено]** Сетевая оптимизация: прорежение феромонов + интерполяция движения на клиенте.
11. **[Выполнено]** Игра по сети (Radmin VPN) + `Открыть порты.bat`.
12. **[Выполнено]** Интерфейс на русском; HUD (склад, рабочие, яйца, личинки, матка, поколение).

## Враги

15. **[Частично]** Умный паук: utility-AI (засада/скрад/погоня/отступление), осмысленное движение без
    искусственных стенок. Своя пищевая цепочка: ест **падаль** (`carrion`) и муравьёв, имеет **логово
    и запас** (таскает падаль в логово → стычки с роем), **гибнет от голода**. Свой геном + эволюция
    охоты (`spider_genome.json`), поколение по смерти/голоду/таймеру → ко-эволюция с колонией.
    Доработать: тюнинг баланса хищник↔рой, другие типы врагов (жуки и т.д.).

## Колониальный ИИ (главное направление)

16. **[Выполнено]** Фундамент: геном (гены с границами), адаптивный контроллер (директивы), фитнес,
    минимальная эволюция (отбор лучшего + мутация при гибели), сохранение в `genome.json` с
    переживанием перезапуска и `reset`.
17. **[Частично]** Тюнинг эволюции — сделано: новая формула фитнеса **без потолка** (еда + убитые
    пауки + средняя популяция + выживание; насыщающийся «интеграл населения» убран), аддитивная
    мутация, архив топ-N + турнирный отбор, счётчик «Поколений», распределение работы (роли/няньки/
    разведка) и казармы. Осталось: тюнинг весов фитнеса и границ генов, баланс хищник↔рой.
18. **[Не начато]** Расширение генома: больше генов (и у колонии, и у паука).
19. **[Частично]** «Зачёт поколения по времени» — СДЕЛАНО: поколение колонии/паука закрывается по
    таймеру (`generationMaxTicks`/`spiderLifeMaxTicks`) ИЛИ по смерти; смерть матки → авто-рестарт;
    эволюция идёт без обязательной гибели; в HUD монотонные счётчики «Поколений (колония/паук)».
    Осталось: график прогресса обучения в HUD.
20. **[Частично]** utility-AI: у паука уже есть; для муравьёв (простая память «последняя еда/паук»,
    utility-выбор действия) — не начато.
21. **[Не начато]** Полноценный ML/RL (обучение с подкреплением) — поздняя веха, «на годы».

## Решение по ИИ — архитектура (справка)

Комбинация автомата и эволюции (согласовано с братом):

- База — FSM у муравья (не «тупой с нуля»): `search → carry → feed → flee` от паука → `dead`.
- Поверх — генетика/эволюция + адаптивный контроллер: «геном» (числа тактики) эволюционирует.
  После гибели — гены лучшего + ~15% мутации.
- Геном сохраняется в JSON и переживает перезапуск сервера.
- Опора на матку/колонию (муравьи роевые).
- НЕ делаем: нейросеть каждому муравью; полноценный ML/RL — отложено «на годы».
