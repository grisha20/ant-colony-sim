# AntColonyAI MVP

Локальный прототип игры-наблюдения про муравьиную колонию: один сервер, две колонии-арены, большая поверхность на весь экран с камерой, подземная сцена с камерами, эволюционирующий ИИ колоний и паука-хищника, WebSocket-снапшоты и сохранение в JSON.

## Что внутри

- `server` — Node.js + TypeScript, авторитетная симуляция с фиксированным тиком 10 раз в секунду.
- `client` — Vite + TypeScript + PixiJS, только рендер снапшотов.
- `shared` — общие типы снапшота мира.
- `snapshot.json` — сохранённое состояние мира.

Подземелье получило слой `diggable underground`: grid клеток `soil/tunnel/chamber/entrance`, список выкопанных комнат, задачи копки и счётчик вынесенной земли `dirtMound`. На чистом старте выкопаны только маточная камера, маленький склад и ход наружу; вход начинается с верхней линии подземелья, а стартовый layout строится от `colonyId`, поэтому гнёзда A/B не совпадают. Камера яиц и ясли не выдаются бесплатно и не сидят в фиксированной точке: колония выбирает свободное место рядом с маткой и ведёт к нему органичный ход по потребности. Комнаты имеют понятную вместимость: одна egg-room держит до 20 яиц, одна nursery — до 20 личинок, один склад — до 100 еды; при переполнении планируется новая отдельная камера, а не раздувание старой в общую пещеру. Рабочие копают доступные соседние клетки, могут помогать на одном фронте и копят землю пачкой (`dirtLoad`, по умолчанию 4 клетки) перед рейсом к входу. На поверхности вход остаётся стабильной дыркой с небольшой воронкой; рост муравейника читается по подземному разрезу, а не по разрастающемуся пятну земли вокруг входа. Комнаты становятся частично пригодными уже после первой выкопанной chamber-клетки, но начатая камера докапывается до чёткой формы.

Поверхность получила декоративный слой: земляной шум, галька, камни, листочки, травинки, трещины и более читаемые входы-норки в стиле SimAnt. Это только рендер и не влияет на симуляцию.

Есть паук-хищник (со своим эволюционирующим ИИ), кормление игроком, эволюция колонии и паука по геному. Намеренно пока нет: базы данных, авторизации, свободного редактора тоннелей, онлайн-мультиплеера, доната и нейросетей (вместо них — генетический алгоритм + utility-AI).

Подробности — в `docs/Документация.md` и `docs/Планы на будущее.md`.

Запуск удобнее через bat-файлы в корне: `Запуск.bat`, `Стоп.bat`, `Рестарт.bat`, `Сброс.bat` (чистый мир), `Сброс обучения.bat` (обнулить геномы), `Открыть порты.bat` (для игры по сети).

## Команды

```bash
npm install
npm run dev
```

Клиент:

```text
http://127.0.0.1:5173
```

WebSocket-сервер:

```text
ws://localhost:8787
```

Чтобы начать с чистой колонии:

```bash
npm run reset
npm run dev
```

`npm run reset` удаляет `snapshot.json`.

## Игра по сети (Radmin VPN)

Второй игрок может открыть твой мир со своего ПК (только наблюдение).

1. На ПК-хосте запусти `npm run dev` (Vite слушает всю сеть, `server.host` включён).
2. В Windows Firewall разреши входящие подключения TCP на портах **5173** (клиент) и **8787** (WebSocket) для Node.js.
3. Подключитесь к одной сети Radmin VPN и узнай свой Radmin-IP (например `26.22.205.122`).
4. Второй игрок открывает в браузере `http://<Radmin-IP-хоста>:5173`.

Клиент сам подключается к WebSocket по тому же хосту, с которого открыта страница
(`ws://<host>:8787`), поэтому отдельно адрес сокета прописывать не нужно. Локально
`http://localhost:5173` продолжает работать как раньше.

## Что проверить

1. При чистом старте колония начинается под землёй: матка, 2 яйца, 4 рабочих, выкопанная маточная камера, маленький склад и ход к поверхности.
2. Поверхность украшена камнями, галькой, листочками, травой и трещинами; рабочие выходят из входа-норки и расходятся наружу.
3. Рабочие идут по запаху еды, забирают еду и несут её на склад под землёй.
4. Еда периодически подрастает и появляется новыми источниками; тела муравьёв остаются падалью, которую можно унести на склад и съесть.
5. Матка откладывает яйца, когда на складе хватает запаса; если свободных клеток egg-room ещё нет, кладка/стресс защищены от стартового death-loop, а при высоком стрессе обычные яйца могут экстренно дорасти у матки в слабых рабочих.
6. Когда у матки есть яйца, колония создаёт задачу на egg-room; когда яйцам нужна стадия личинки, копает nursery. Рабочие выносят грунт к входу пачками, а няньки переносят яйца после появления свободной выкопанной клетки яйцекамеры.
7. Яйца-принцессы под стрессом не превращаются в слабых рабочих: они пополняют банк принцесс и могут унаследовать трон при смерти матки.
8. Переключатель `Поверхность` / `Подземелье` меняет вид между картой сверху и боковым срезом муравейника; техническая разметка планируемых клеток копки скрыта по умолчанию.

Баланс лежит в `server/src/config.ts`.
